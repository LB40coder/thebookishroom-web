# Security — The Bookish Room Admin & API

## Overview

The admin area and API use **defense in depth**. No single layer is "unhackable", but together they make unauthorized access extremely difficult.

## Environment Variables

Run `npm run generate-secrets` to create secure values:

| Variable | Purpose |
|----------|---------|
| `ADMIN_PATH` | Secret URL path (e.g. `cms-a1b2c3d4...`) — **never share publicly** |
| `ADMIN_API_KEY` | Bearer token for your bot |
| `SESSION_SECRET` | Signs admin session cookies |
| `ADMIN_PASSWORD_HASH` | Bcrypt hash of your admin password |
| `DATABASE_URL` | PostgreSQL connection string |

## Admin Access (Human)

1. Go to `https://yoursite.com/{ADMIN_PATH}/login`
2. Enter your password
3. Session lasts 8 hours (httpOnly, secure, sameSite=strict cookie)

**Security measures:**
- Admin lives at a **secret path** — `/admin` does not exist
- Direct access to `/_studio` returns **404**
- Login rate-limited: **5 attempts per 15 minutes** per IP
- Password stored as **bcrypt hash** (never plain text)
- Generic error messages (no user enumeration)

## Bot API

Base URL: `https://yoursite.com/api/v1`

All requests require:

```
Authorization: Bearer YOUR_ADMIN_API_KEY
Content-Type: application/json
```

### Posts

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/posts` | List all posts |
| GET | `/api/v1/posts/{id-or-slug}` | Get one post |
| POST | `/api/v1/posts` | Create post |
| PUT | `/api/v1/posts/{id-or-slug}` | Update post |
| DELETE | `/api/v1/posts/{id-or-slug}` | Delete post |

### Books

| Method | Endpoint |
|--------|----------|
| GET/POST | `/api/v1/books` |
| GET/PUT/DELETE | `/api/v1/books/{id-or-slug}` |

### Authors

| Method | Endpoint |
|--------|----------|
| GET/POST | `/api/v1/authors` |
| GET/PUT/DELETE | `/api/v1/authors/{id-or-slug}` |

### Example — Create a post

```bash
curl -X POST https://yoursite.com/api/v1/posts \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "10 Books That Feel Like Dark Academia",
    "slug": "books-that-feel-like-dark-academia",
    "excerpt": "From secret societies to gothic libraries...",
    "coverImage": "/images/gothic.svg",
    "seoTitle": "10 Books That Feel Like Dark Academia | The Bookish Room",
    "seoDescription": "The best dark academia books...",
    "tags": ["dark academia", "gothic"],
    "moods": ["dark-academia", "gothic"],
    "published": true
  }'
```

## Security Layers

1. **Secret admin path** — unknown URL = invisible admin
2. **API key authentication** — timing-safe comparison, min 32 chars
3. **Session JWT** — signed with HS256, httpOnly cookie
4. **Rate limiting** — login and API write endpoints
5. **Input validation** — Zod schemas on all inputs
6. **API request logging** — `ApiLog` table for audit trail
7. **No admin links** on public site
8. **robots.txt** blocks `/api/` and `/_studio`

## Deployment Checklist (Vercel)

1. Create Vercel Postgres database
2. Set all env vars in Vercel dashboard
3. Run `npx prisma db push` (or add to build script)
4. **Never** commit `.env.local` or API keys to GitHub
5. Rotate `ADMIN_API_KEY` if compromised
6. Use HTTPS only (Vercel default)

## Important

- No system is 100% impossible to breach
- Keep dependencies updated
- Use strong, unique secrets (generated by the script)
- Consider Vercel WAF / Upstash Redis rate limiting for production scale
